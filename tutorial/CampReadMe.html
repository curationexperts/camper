Samvera Tutorial Resources

We've assembled the resources listed below as your jumping off point for various Samvera tutorials.

Contents

  • Samvera Camp Syllabus - the syllabus from the Fall 2018 Camp in Durham, NC with running updates for use during camp
  • Sample Assets - sample files for use while completing the various hydra tutorials
  • Vagrantfile - a configuration file for Vagrant which can be used to download and start a pre-configured virtual machine with all the software installed that you need to successfully complete a variety of Hydra tutorials

Using the Virtual Machine

  • Install the required software

    • Install the current version of VirtualBox for your system from the provided thumbdrive or from https://www.virtualbox.org/wiki/Downloads
    • Install the current version of Vagrant for your system from the provided thumbdrive or from https://www.vagrantup.com/downloads.html
    • Windows Users only need to install an ssh client, we've had sucess with the version provided as part of the Github Desktop

  • Setting up the tutorial VM from the web

    • Open a terminal window on your system
    • Create a new directory to run the VM from by typing the following commads
      NOTE: some of the tutorial steps may assume you're working in a directory named "/vagrant"'
      mkdir vagrant
      cd vagrant
    • Get a copy of the Vagrantfile we've provided
      wget http://camp.curationexperts.com/Vagrantfile

  • Setting up the tutorial VM from a thumb-drive (in class)

    • Copy the /tutorial directory from the thumb-drive to your hard drive
    • Open a terminal window on your system
    • Make the tutorial directory your working directory
      cd ~/<your-local-desitination>/tutorial
    • We've already provided a copy of the Vagrant file if you've copied from the thumbdrive

  • Running Vagrant

    • Make sure you're starting from your tutorial directory with your Vagrantfile in it
    • Start vagrant by typing vagrant up
      The first time you do this, it may take some time to download the initial VM file
    • Log in to the VM by typing vagrant ssh
      You should get a command prompt something like vagrant@camper:~$
    • Log out of the VM by typing exit
    • Stop vagrant by typing vagrant halt

  • The machine is configured with a user named 'vagrant' and password set to 'vagrant'.  The virtual machine uses NAT to access network resources, but port forwarding has been setup to allow you to view the Rails development server, Solr, and Fedora from your main browser.

    For performanace and compatabilty with most systems, the VM is configured to use only 2048MB of memory and 2 CPUs.  If your system has significantly more or less RAM installed, you may want to adjust the memory allocation accordingly. You can do this by editing the Vagrantfile in the tutorial directory.

    NOTES: The system is configured to facilitate development and most security settings are left relatively open - DO NOT USE THIS VM FOR PRODUCTION PURPOSES
